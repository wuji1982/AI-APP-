- API 层只负责路由声明与参数校验，所有业务规则下沉到 `app/services/*_service.py` 中的静态方法或类方法，避免在路由函数内写 SQL。
- 数据库访问统一通过 `app.database.get_db` 依赖注入 `AsyncSession`，并在 try/except 中 commit/rollback/close，禁止手动管理会话生命周期。
- 业务常量（拼团倍数、分润比例、贡献值利率、门店阶梯等）集中定义在 `app.config.Settings` 类字段中，Service 一律通过 `settings.XXX` 读取，不在代码里硬编码数值。
- 异步任务按业务域拆分为独立文件（`*_tasks.py`），统一在 `tasks/__init__.py` 中导入以便 Celery autodiscover，broker/result backend 由 `celery_app.py` 集中配置。
- AI Agent 遵循 `BaseAgent` 抽象接口，子类必须实现 `execute` 与 `should_continue`，并通过 `run` 包装统一的日志与异常返回格式。
