本项目采用纯 Docker Compose 驱动的容器化构建与部署方案，未使用 Makefile、CI/CD 流水线或独立打包脚本。核心构建与编排逻辑集中在三个文件中：

1. 后端镜像构建（backend/Dockerfile）
- 基于 python:3.11-slim 基础镜像，WORKDIR 为 /app
- 依赖安装通过清华源加速：pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
- 开发模式启动：uvicorn app.main:app --reload，暴露 8000 端口
- 无多阶段构建、无缓存层优化、无 .dockerignore

2. 服务编排（docker-compose.yml）
- 统一编排 7 个服务：PostgreSQL 16、Redis 7、RabbitMQ 3-management、MinIO、FastAPI 后端、Celery Worker、Celery Beat、Nginx
- 使用 depends_on + healthcheck 控制启动顺序（仅 PostgreSQL 配置了健康检查）
- 环境变量集中注入数据库、Redis、Celery Broker/Backend 连接串
- 后端源码通过 volume 挂载到容器实现热重载
- 数据持久化通过 named volumes（pgdata、redisdata、miniodata）

3. 反向代理（nginx.conf）
- Nginx 作为入口，监听 80 端口
- /api/ 路径反向代理至 backend:8000
- 预留 /ws/ WebSocket 代理和静态资源目录
- 通过 volume 将宿主机 nginx.conf 以只读方式映射到容器

4. 依赖管理（backend/requirements.txt）
- 固定版本锁定所有 Python 依赖，包括 FastAPI、SQLAlchemy、Celery、LangChain/LangGraph、OpenAI SDK 等
- 未使用 pipenv/poetry/conda 等现代工具链

缺失的构建能力：
- 无前端构建流程（frontend/web-admin 和 mobile-app 目录下未见 package.json、Dockerfile 或构建脚本）
- 无 CI/CD 配置文件（无 .github/workflows、Jenkinsfile、.gitlab-ci.yml 等）
- 无发布/版本管理脚本
- 无测试执行命令集成
- 无生产环境镜像优化（如多阶段构建、非 root 用户运行）
